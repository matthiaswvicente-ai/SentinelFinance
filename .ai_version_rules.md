# DIRETRIZES DE ATUALIZAÇÃO E SYNC DE VERSÕES (EXCLUSIVO DESENVOLVEDOR / IA)

Este arquivo contém instruções críticas que toda Inteligência Artificial ou Desenvolvedor deve ler e verificar antes de criar, empacotar ou subir qualquer atualização para o repositório do GitHub. Isso evita a desconfiguração e a perda de sincronia entre o código modificado localmente e a versão distribuída aos usuários.

---

## 1. Verificações Obrigatórias Pré-Release

Antes de disponibilizar uma nova versão no GitHub:
1. **Incremento de Versão:** A versão do programa deve ser incrementada em:
   - `update_manager.py` (propriedade `self.current_version` ou variável equivalente).
   - O arquivo `version.json` hospedado se houver.
   - A tag de release do GitHub correspondente (ex: `v1.1.0`).
2. **Verificação de Retrocompatibilidade do Banco de Dados:**
   - Nunca modifique ou remova colunas existentes de tabelas cruciais (`Transacoes`, `Categorias`, `Preferencias`, `Cartoes`, `Carteira_Investimentos`) sem fornecer scripts de migração automática no método `create_tables()` ou equivalente.
   - Qualquer nova coluna inserida no schema padrão deve ser adicionada via cláusula `ALTER TABLE ... ADD COLUMN ...` protegida por blocos `try/except sqlite3.OperationalError` para evitar falhas em bancos de dados antigos.
3. **Preservação de Dados do Usuário:**
   - O processo de atualização automática **nunca** deve substituir ou deletar os arquivos:
     - `*.db` (bancos de dados do usuário: `financas.db`, `sandbox_financas.db`, etc.)
     - Pasta `backups/` e backups automáticos.
     - Logs locais na pasta `logs/`.

---

## 2. Fluxo do Mecanismo de Auto-Update

O script de atualização automática em `update_manager.py` realiza o seguinte procedimento:
1. **Requisição à API:** Consulta `https://api.github.com/repos/Matthiaswvicente-ai/SentinelFinance/releases/latest` com um cabeçalho de User-Agent apropriado.
2. **Comparação de Versão:** Extrai `tag_name` (ex: `v1.1.0`) e compara com a versão local.
3. **Download Temporário:** Baixa o código-fonte compactado do zipball para a pasta temporária de sistema (`tempfile.gettempdir()`).
4. **Extração Seletiva e Sobrescrita:**
   - Descompacta os arquivos.
   - Copia recursivamente os arquivos `.py` e assets do repositório para o diretório atual do aplicativo.
   - Ignora estritamente qualquer arquivo com extensão `.db` ou que esteja no diretório `backups/`.
5. **Notificação de Reinício:** Solicita o reinício do processo para recarregar o interpretador Python com o novo código.

---

## 3. Prevenção de Desassociação (Desincronização)

- Ao subir alterações locais, certifique-se de que os arquivos principais (`main.py`, `database.py`, `app_ui.py`, `v2/main_v2.py`, `v2/database.py`, `update_manager.py`) estejam com as dependências do `requirements.txt` atualizadas.
- Se uma biblioteca nova for adicionada, ela deve ser listada em `requirements.txt`.
